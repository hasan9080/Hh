Clusters Kubernetes

O Kubernetes coordena um cluster de computadores altamente disponível que estão interligados para trabalhar como uma única unidade. As abstrações no Kubernetes permitem-lhe implementar aplicações contentorizadas num cluster sem as vincular especificamente a máquinas individuais. Para tirar partido deste novo modelo de implementação, as aplicações precisam de ser empacotadas de forma a desacoplá-las de hospedeiros individuais: precisam de ser contentorizadas. As aplicações contentorizadas são mais flexíveis e disponíveis do que nos modelos de implementação anteriores, onde as aplicações eram instaladas diretamente em máquinas específicas como pacotes profundamente integrados no hospedeiro. O Kubernetes automatiza a distribuição e agendamento de contentores de aplicações através de um cluster de uma forma mais eficiente. O Kubernetes é uma plataforma de código aberto e está pronta para produção.

Um cluster Kubernetes consiste em dois tipos de recursos:

  • O Plano de Controlo coordena o cluster
  • Nós são os trabalhadores que executam as aplicações

Diagrama do Cluster


O Plano de Controlo é responsável por gerir o cluster. O Plano de Controlo coordena todas as atividades no seu cluster, como o agendamento de aplicações, manutenção do estado desejado das aplicações, escalonamento das aplicações e lançamento de novas atualizações.

Um nó é uma VM ou um computador físico que serve como uma máquina trabalhadora num cluster Kubernetes. Cada nó tem um Kubelet, que é um agente para gerir o nó e comunicar com o plano de controlo Kubernetes. O nó também deve ter ferramentas para lidar com operações de contentores, como o {{< glossary_tooltip text="containerd" term_id="containerd" >}} ou o {{< glossary_tooltip term_id="cri-o" >}}. Um cluster Kubernetes que lida com tráfego de produção deve ter um mínimo de três nós porque se um nó falhar, perde-se tanto um membro do etcd quanto uma instância do plano de controlo, e a redundância fica comprometida. Pode mitigar este risco adicionando mais nós ao plano de controlo.

Os Planos de Controlo gerem o cluster e os nós que são usados para hospedar as aplicações em execução.

Quando implementa aplicações no Kubernetes, indica ao plano de controlo para iniciar os contentores de aplicações. O plano de controlo agenda os contentores para serem executados nos nós do cluster. Componentes a nível de nó, como o kubelet, comunicam com o plano de controlo usando a API do Kubernetes, que o plano de controlo expõe. Os utilizadores finais também podem usar a API do Kubernetes diretamente para interagir com o cluster.

Um cluster Kubernetes pode ser implementado tanto em máquinas físicas como virtuais. Para começar com o desenvolvimento Kubernetes, pode usar o Minikube. O Minikube é uma implementação leve do Kubernetes que cria uma VM na sua máquina local e implementa um cluster simples contendo apenas um nó. O Minikube está disponível para sistemas Linux, macOS e Windows. A CLI do Minikube fornece operações básicas de inicialização para trabalhar com o seu cluster, incluindo iniciar, parar, estado e eliminar.

Agora que sabe mais sobre o que é o Kubernetes, visite Olá Minikube para experimentar isto no seu computador.
